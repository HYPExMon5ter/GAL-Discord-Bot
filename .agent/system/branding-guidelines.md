# Branding Guidelines

## Overview
This document establishes the visual identity and branding standards for the Guardian Angel League Live Graphics Dashboard and Discord Bot ecosystem.

## Visual Identity

### Logo and Brand Elements
- **Primary Logo**: Guardian Angel League emblem
- **Color Palette**: Discord-inspired with custom accent colors
- **Typography**: Modern, clean sans-serif fonts
- **Iconography**: Consistent with Discord/React design systems

### Color System
#### Primary Colors
- **Primary Blue**: `#5865F2` (Discord brand blue)
- **Secondary Blue**: `#4752C4`
- **Accent Green**: `#3BA55C` (Success states)
- **Accent Red**: `#ED4245` (Error states)

#### Neutral Colors
- **Background**: `#36393F` (Dark theme)
- **Surface**: `#2F3136`
- **Text Primary**: `#DCDDDE`
- **Text Secondary**: `#B9BBBE`
- **Text Muted**: `#72767D`

### Typography
#### Font Hierarchy
- **Headings**: Bold, larger font sizes
- **Body Text**: Regular weight, readable sizes
- **UI Elements**: Consistent with system fonts

#### Accessibility Standards
- **Contrast Ratios**: Minimum 4.5:1 for normal text
- **Screen Reader Support**: Proper ARIA labels
- **Keyboard Navigation**: Clear focus indicators

## UI Component Standards

### Dashboard Components
- **Cards**: Consistent shadow, border radius, spacing
- **Buttons**: Uniform hover states, loading indicators
- **Forms**: Clear validation, error messaging
- **Tables**: Consistent alignment, sorting indicators

### Discord Integration
- **Embeds**: Match Discord embed styling
- **Buttons**: Follow Discord button patterns
- **Colors**: Use Discord's color palette where appropriate

## Content Guidelines

### Tone of Voice
- **Professional**: Clear, concise communication
- **Helpful**: User-focused language
- **Consistent**: Unified terminology across all interfaces

### Text Standards
- **Labels**: Clear, descriptive, action-oriented
- **Messages**: Contextual, informative, timely
- **Error Messages**: Helpful, specific, solution-oriented

## Implementation Guidelines

### React Components
- **Design System**: shadcn/ui components with custom theming
- **CSS Variables**: Centralized color and spacing definitions
- **Responsive Design**: Mobile-first approach

### API Integration
- **Loading States**: Consistent loading indicators
- **Error Handling**: User-friendly error messages
- **Success Feedback**: Clear confirmation of actions
